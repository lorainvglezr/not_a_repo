Preguntas a responder...

  1. ¿Qué diferencias y semejanzas hay entre Java y JavaScript?
    2. A pesar del pensamiento contrario de la mayoría de las personas, Java y JavaScript son dos lenguajes de programación diferentes que comparten cierta parte de sus respectivos nombres. Primero que nada, Java es un lenguaje de programación orientado a objetos o multiproceso compilado que se utiliza principalmente en desarrollo de aplicaciones móviles, aplicaciones de servidor y sistemas empresariales (Java Software, s. f.). Asimismo, cuando hablamos de JavaScript nos referimos a un lenguaje de programación interpretado, lo que quiere decir esto de “interpretado” es que se interpreta el código mientras se ejecuta; JavaScript se usa mayoritariamente para generar páginas interactivas que permitan el dinamismo de las mismas con el usuario. Finalmente, dejando de lado las diferencias mencionadas anteriormente, conocemos que su principal similitud se basa en la sintaxis, así como también ambos lenguajes son primordiales para el desarrollo de software moderno y que se pueda interactuar en entornos web, donde Java puede ser utilizado en el backend y JavaScript en el frontend (JavaScript | MDN, s. f.).

  2. ¿Qué métodos tiene el objeto Date? (Menciona al menos 5*)
    3. El objeto “Date” en JavaScript proporciona diversos métodos para poder trabajar con fechas y horas específicas. En general, estos métodos permiten la manipulación y el correcto formateo de fechas de manera efectiva en aplicaciones web. Entre los métodos más utilizados se encuentras:
    • getDate(): Este se encarga de devolver el día del mes.
    • getMonth(): Es el encargado de retornar el mes y va de un rango de 0 a 11.
    • getFullYear(): Se caracteriza por proporcionar el año completo.
    • getTime(): Este devuelve el tiempo en milisegundos desde el 1 de enero de 1970.
    • getHours(): Se encarga de devolver la hora de la fecha en formato de 24 horas.

    (Date - JavaScript | MDN, 2024)


  3. ¿Qué métodos tienen los arreglos? (Menciona al menos 5*)
    4. Cuando hablamos de arreglos en JavaScript, se sabe que estos cuenta con una alta y variada gama de métodos que simplifican su manipulación y gestión. Dichos métodos autorizan la realización de operaciones de manera eficaz y precisa en los arreglos a manipular. Algunos de los más comunes son:
    • push() Este método se encarga de añadir uno o más elementos al final de un arreglo.
    • pop(): Básicamente lo que hace es eliminar el último elemento de un arreglo.
    • shift(): Se caracteriza por eliminar el primer elemento que se encuentre dentro del arreglo.
    • unshift(): Este método agrega uno o más elementos al inicio del arreglo.
    • map(): Genera un nuevo arreglo con los resultados de aplicar una función a cada elemento del arreglo original.

    (Array - JavaScript | MDN, s. f.)


  4. ¿Cómo se declara una variable con alcance local dentro de una función?
    5. En términos de JavaScript, para iniciar el proceso de declarar una variable con alcance local dentro de una función se utiliza la palabra reservada “let” que es aquella que permite que la variable sea accesible únicamente dentro del bloque de la función (Let - JavaScript | MDN, s. f.). Sin embargo, si lo que se desea es que la variable no cambie el valor que le fue asignado al inicio, lo que se debe de utilizar es otra palabra reservada llamada “const” que es aquella que garantiza que el valor no sea modificado, pero también limita el alcance de la variable a la función (Const - JavaScript | MDN, s. f.). En resumen, “let” y “const” proporcionan un control eficaz y práctico acerca del alcance y la versatilidad de las variables en el contexto de funciones.

  5. ¿Qué implicaciones tiene utilizar variables globales dentro de funciones?
    6. Al utilizar variables globales dentro de las funciones de JavaScript, nos podemos llegar a topar con distintas y diversas implicaciones negativas. Primero que nada, esto aumenta el riesgo de llevar a cabo una sobreescritura accidental por el simple hecho de que cualquier parte del código es capaz de modificar el valor de la variable global, y esto generaría errores difíciles de rastrear (Difference Between Variable Declaration Syntaxes In Javascript (Including Global Variables)?, s. f.). De igual manera, ya que las variables globales son accesibles desde cualquier parte del código, esto puede dificultar la gestión del estado de la aplicación y generar que el código persé sea menos modular y más propenso a dificultades. En última instancia, el uso excesivo o desmesurado de estas variables puede influir en el rendimiento general porque el motor de JavaScript debe de gestionar una cantidad mayor de número de referencias y accesos a estas variables, lo que puede enlentecer la ejecución del código (W3Schools.com, s. f.).
