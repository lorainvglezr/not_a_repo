LABORATORIO 1

Preguntas a responder...

  1. ¿Cuál es la diferencia entre Internet y la World Wide Web?
    2. Al hablar de internet nos referimos a la red que conecta las computadoras alrededor del mundo. Mientras que la World Wide Web es la colección de páginas o documentos que se asientan en el internet. En resumen, para acceder a la web, es necesario el uso del internet.
    (Redacción, 2019).

  2. ¿Cuáles son las partes de una URL?
    3. Una URL suele estar conformada por 3 o 4 componentes, los cuales son:
    • Un esquema: Es el protocolo para acceder al recurso en internet.
    • Un nombre de host: Pueden ir seguidos de un número de puerto.
    • Un camino: La ruta que identifica el recurso.
    • Una cadena de consulta: Ésta sigue al componente de ruta y proporciona una cadena de información.
    (IBM Documentation, s.f.).

  3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
    4. Los métodos HTTP son tácticas para pedir información particular. Existen varios métodos específicos con funciones diversas y completas, tales como:
    • El método GET: Las peticiones sólo deben de recuperar datos.
    • El método HEAD: Solicita una respuesta exactamente igual a la de una petición GET, pero no incluye el cuerpo de la respuesta.
    • El método POST: Envía una entidad o un dato nuevo a un recurso específico.
    • El método PUT: Reemplaza lo solicitado en un recurso en específico.
    • El método PATCH: Actualiza un elemento de forma parcial.
    • El método DELETE: Elimina un elemento o dato específico.
    (MDN Web Docs, 2023).

  4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?
    5. Desde mi punto de vista, es necesario utilizar el método POST dado que yo, como cliente, deseo ingresar un dato nuevo y no antes registrado al sitio web.

  5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?
    6. Yo opino que sería necesario mandar a llamar todo lo que el sitio contiene de cualquier lugar en el que se encuentre y si este es el caso, lo más óptimo sería usar el método GET.

  6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?
    7. Al recibir como respuesta de un servidor web un código 200 no nos indica que hubo algún error, es todo lo contrario, ya que este código 200 nos dice que se recibió, procesó y realizó de manera correcta y sin problemas la solicitud enviada.
    (dotcom-monitor, 2021).

  7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?
    8. No, no es responsabilidad del desarrollador corregir nada debido a que este error 404 simplemente significa que no se logra encontrar en el servidor del sitio web la página que fue solicitada por el usuario por la falta de existencia de este. O, en su defensa, lo que sucede es que el usuario intenta acceder a un recurso prohibido y por razones de seguridad se le indica al usuario que el recurso no fue encontrado.
    (Boada, 2025).

  8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?
    9. Sí, es responsabilidad del desarrollador corregir un sitio web que reporta un error 500 dado que este tipo de errores aparecen porque el servidor no logró cumplir con la solicitud del cliente debido a que se encontró con una condición inesperada o un error interno que puede tener diferentes orígenes, esto nos indica que hubo algo que el desarrollador no consideró y es su labor corregir dicha situación incorrecta. Es importante mencionar que este suele ser el peor error que puede aparecer por las infinitas causas posibles con las que cuenta.
    (Sanjurjo, 2024).

  9. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.
    10. Cuando decimos que un atributo está despreciado o desaprobado lo que se intenta decir es que, gracias a la constante evolución de HTML5, existen etiquetas que se volvieron obsoletas. En otras palabras, estos son elementos que aún se pueden emplear y utilizar, pero se recomienda no hacerlo, ya que los navegadores podrían dejar de darles soporte en poco tiempo, lo que ocasionaría una mala o incorrecta visualización de la página persé. Algunos elementos de HTML4 que en HTML5 están desaprobados son:
    • bgsound: Servía para insertar sonidos de fondo.
    • basefont: Se utilizaba para determinar la tipografía estándar que se enseñaría en la página web.
    • center: Funcionaba para indicarle al lenguaje que centre el texto.
    • menu: Generaba menús de listados o viñetas.
    (Tokio School, s.f.).

  10. ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?
    11. Existen varias diferencias entre HTML4 y HTML5, algunas de las más destacadas y principales son las siguientes:
    • Compatibilidad del navegador: Mientras que HTML5 es compatible con las versiones de navegadores más recientes como Google Chrome, Firefox, Safari, Opera e Internet Explorer 9 o superior, HTML4 se limita solo a la compatibilidad con navegadores antiguos y modernos. Es importante destacar que las especificaciones de HTML5 no pueden ser procesadas por navegadores antiguos como Internet Explorer 8 o versiones anteriores.
    • Soporte multimedia: Otra gran diferencia entre estas dos versiones del HyperText Markup Language es el soporte multimedia. Este gran punto resalta que el HTML5 permite agregar audio y video a un sitio web sin la necesidad de complementos adicionales, así como también hace posible el dibujar gráficos y animaciones en tiempo real; sin embargo HTML4 era un poco más limitado en este ámbito.
    • Formularios mejorados: Se han mejorado los elementos de formulario existentes y se han agregado algunos nuevos a HTML5, esto debido a que es una parte fundamental de un sitio web que la entrada de datos para los usuarios sea lo más sencilla posible, lo que facilita y mejora la interacción de los usuarios con el sitio web en comparación con la versión anterior, HTML4.
    (desarrolladoraweb, s.f.).

  11. ¿Qué componentes de estructura y estilo tiene una tabla?
    12. En cuanto a aspectos de estructura, podemos encontrar características técnicas o de formato que debe de tener una tabla, tales como encabezados de columna, columnas, filas, leyenda, cuerpo, pie de tabla, celda, entre otras cosas. Mientras que si hablamos de estilo, podemos incluir particularidades más relacionadas al diseño de la página web como el color de borde, color de celda, ancho, relleno y mucho más.
    (Marco, s.f.).

  12. ¿Cuáles son los principales controles de una forma HTML5?
    13. Los principales controles de una forma en HTML5 son los botones, controles de fecha, controles de hora, campos de texto y campos de validación.
    (MDN Web Docs, s.f.).

  13. ¿Qué tanto soporte HTML5 tiene el navegador que utilizas?Puedes utilizar la siguiente página para descubrirlo: http://html5test.com/ (Al responder la pregunta recuerda poner el navegador que utilizas)
    14. Google Chrome, el cual es el navegador que yo utilizo con más frecuencia, tiene un excelente soporte para HTML5, ya que cuenta con una casi total compatibilidad con las últimas tecnologías y características web. Al visitar http://html5test.com/ desde Chrome, se obtiene una puntuación de 532 de un total de 555 puntos, lo que indica un alto nivel de compatibilidad.
    (HTML5test, s.f.).

  14. Sobre el ciclo de vida y desarrollo de los sistemas de información:
    • ¿Cuál es el ciclo de vida de los sistemas de información?
      • El ciclo de vida de un sistema de información abarca todas las fases desde la concepción hasta el retiro del sistema. Las fases principales del ciclo de vida de los SISs son la planificación, el análisis, el diseño, el desarrollo, las pruebas, la implementación, el mantenimiento y el retiro.
      (Euroinnova International Online Education, 2024).
    • ¿Cuál es el ciclo de desarrollo de sistemas de información?
      • El ciclo de desarrollo de sistemas de información se enfoca en la creación e implementación del software. Las fases características del ciclo de desarrollo de SISs incluyen la recolección de requisitos, la planificación y el análisis, el diseño, la codificación, las pruebas, la implementación y el mantenimiento.
      (Amazon Web Services, s.f.).
