Preguntas a responder...

  1. Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?
    2. Como ingeniero de software, primero mencionaría que el uso del “!important” en CSS debe de ser limitado y considerado como último recurso, ya que este comando puede dificultar el mantenimiento y la legibilidad del código. Yo recomendaría que primero se identifiquen y clasifiquen las prioridades dentro del programa a realizar, así como también sería de gran ayuda el revisar la estructura de nuestro CSS para intentar mejorar la organización y tener la posibilidad de reducir la dependencia de “!important”. En conclusión, no hay que sobreusar este comando para lograr obtener un código limpio y mantenible, con el fin de únicamente enmarcar aquellos atributos que necesiten un especial enfoque en donde se vea prudente.

  2. Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?
    3. Se debe de tener cuidado al escoger una imagen de fondo en una página HTML, ya que ésta puede afectar en el rendimiento y la legibilidad del sitio web, así como también podría influir en la coherencia estética del contenido en general. Es sumamente importante que la imagen se encuentre en un punto medio en cuanto a claroscuros, lo que quiere decir que es fundamental considerar que la imagen no debe de ser ni tan oscura ni tan brillante con el fin de facilitar la lectura del texto, así como también es necesario que la imagen tenga el tamaño adecuado para evitar que se pixelee y se aumente el peso de la página web. Además de todo lo anterior, es sustancial considerar la responsividad, ya que una imagen que se ve bien en pantallas grandes puede no funcionar en dispositivos móviles. En última instancia, hay que tener en cuenta cuestiones de accesibilidad, como no utilizar imágenes con información importante, ya que los lectores de pantallas no las pueden leer. En pocas palabras, una buena elección de imagen ofrece una mejor experiencia para el usuario y un buen diseño final de la página web.

  3. Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?
    4. Yo recomiendo que, al elegir las unidades de una propiedad de estilo en CSS, se considere el contexto y el propósito del diseño a implementar. Si tomamos en cuenta que las unidades en píxeles (px) y puntos (pt) son fijas, podemos deducir que independientemente del tamaño de la pantalla, los elementos con estas características se mantendrán con su tamaño exacto, lo cual puede afectar de forma significativa en la responsividad de la página web. Mientras que, si hablamos de las unidades porcentuales (%), se podría decir que estas son las ideales o las más adecuadas para generar diseños flexibles y adaptativos, ya que posibilitan que los elementos escalen en concordancia con su contenedor y esto simplifica la adaptación de la página web a distintos tamaños en la pantalla. Por otro lado, yo recomendaría más utilizar las unidades relativas como “em” o “rem” para el texto en entornos de impresión, en vez de las unidades en puntos, dado que aumentan la accesibilidad y la escalabilidad. En definitiva, mi recomendación final sería hacer uso de las unidades de una propiedad de estilo porcentual y también utilizar las unidades relativas para generar un diseño más reservado y accesible, dejando el uso de los píxeles para casos específicos donde sí se necesite esta aproximación.

  4. ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?
    5. El uso de una versión minimizada del CSS mejora el rendimiento del sitio, dado que reduce el tamaño final del archivo al eliminar espacios en blanco, así como también elimina comentarios o caracteres innecesarios; esto permite que el archivo pese menos y cargue de forma más rápida y eficaz. Asimismo, minimizar el CSS no sólo mejora la experiencia del usuario, sino que también ayuda a que en conexiones lentas se optimice el uso del ancho de banda y se tenga una mejor puntuación en métricas de rendimiento, lo que influencia positivamente en la Optimización para Motores de Búsqueda del sitio. En resumidas cuentas, la minimización del CSS acelera la carga de la página y mejora la eficiencia general del sitio web.
